## Critical architectural facts (already confirmed, don't re-derive)

1. **No envelope.** `httpClient.get<T>()` does `response.json() as T` directly ‚Äî the frontend wants the raw object body, **not** our `{success, data, message, timestamp}` envelope that every `/api/v1/*` endpoint uses.
2. **Route prefix.** Frontend calls `/api/...` (no `/v1`). Base URL comes from `VITE_API_BASE_URL`.
3. **camelCase.** All JSON keys. Backend is snake_case everywhere else.
4. **Auth response shape differs entirely** from our real `/api/v1/auth/login` (`{access_token, refresh_token, user: {id, email, full_name, role, ...}}`). Contract wants `{token, user: {name, email, workspace}}`.
5. **Resolution:** a brand-new adapter layer, **not** a modification of `/api/v1/*`. New routers mounted directly on the FastAPI app (not under `api_router`) at `/api/*`, returning camelCase via a shared base model `app/schemas/camel.py` (`CamelModel`, already created ‚Äî uses `pydantic.alias_generators.to_camel` + `populate_by_name=True`; FastAPI serializes aliases by default so just use `response_model=YourCamelModel` and return normal snake_case-populated Python objects).
6. CORS origins updated to include `http://localhost:5173` (Vite default) in `.env`, `.env.example`, and `app/config/settings.py` default ‚Äî **done**.
